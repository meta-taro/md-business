# ログイン機能 検証シート（サンプル）

> v0.7.1 動作確認用のサンプル。Google Workspace アドオンのサイドバーで本ファイルを textarea に貼り付けて「検証シート: セットアップ」を押すと、現在の Sheet にスキーマ駆動の表（DataValidation プルダウン + ConditionalFormat 行背景色 + DateValidation 日付ピッカー）が作成されます。

## 1. 機能概要

ログイン画面における基本的な認証フローと、エラーパターンに対するシステム挙動を確認する。

## 3. 想定使い方

1. Google Sheets で新規シートを作成し、サイドバーで md-business を開く
2. 本ファイル全体（frontmatter 含む）をコピー → サイドバーの textarea に貼り付け
3. 「検証シート: セットアップ」ボタン → Sheet 1 行目にヘッダー + プルダウン / 行背景色 / 日付ピッカーが自動適用される
4. 担当者がブラウザ上で「結果 / 担当 / 実施日 / 備考」を埋める
5. 「md 書き出し」ボタン → サイドバーに編集後の Markdown が表示される

## 4. Sheets ⇆ GitHub 同期（v0.7.1 〜）

- frontmatter の `シートID:` に当該 Google Sheets のファイル ID を入れる → 当該 Sheet とリンク
- frontmatter の `repository: your-name/your-repo@main:docs/specs/login-spec.md` を **自分の repo に書き換え**（push 先の宣言）
- サイドバーで PAT（contents: write 権限）を保存
- Sheet を編集後、サイドバーの **「GitHub に push」ボタン** を押下 → GitHub Contents API 経由で md ファイルに commit（`git push` と同じメンタルモデル）
- 自動同期は v0.7.1 で廃止（編集中の中間状態で git 履歴が汚れるのを避けるため）
