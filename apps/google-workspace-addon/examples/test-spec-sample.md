# ログイン機能 検証シート（サンプル）

> Phase C 動作確認用のサンプル。Google Workspace アドオンのサイドバーで以下の Markdown テーブルを貼り付けて「インポート」を押すと、現在の Sheet に書き込まれる。書き込み後にセルを編集 → 「エクスポート」を押すと、編集結果がサイドバーに Markdown として戻ってくる。

## 1. 機能概要

ログイン画面における基本的な認証フローと、エラーパターンに対するシステム挙動を確認する。

## 2. 検証項目

| ID | 検証項目 | OK/NG | 担当 | 実施日 | 備考 |
|---|---|---|---|---|---|
| T-001 | 正しい ID / パスワードでログイン成功 |  |  |  |  |
| T-002 | 誤ったパスワードでエラーメッセージ表示 |  |  |  |  |

| T-010 | 「ログアウト」ボタンで即時ログアウト |  |  |  |  |

## 3. 想定使い方

1. このファイルを Google Sheets で新規作成し、サイドバーで md-business を開く
2. 上記 `## 2. 検証項目` の表をコピー → サイドバー textarea に貼り付け
3. 「インポート」ボタン → Sheet 1 行目から書き込まれる
4. 担当者がブラウザ上で OK/NG / 担当 / 実施日 / 備考 を埋める
5. 完了したら「エクスポート」→ サイドバーに編集後の Markdown が表示
6. それを `.md` にコピペして git commit

## 4. 双方向同期の理想形（v0.2.0 〜）

- frontmatter に `googleSheetId: <ID>` を書く → 自動的に当該 Sheet とリンク
- Sheet 編集 → onEdit trigger → GitHub API へ commit
- `.md` 編集 → CI で Sheet を上書き
- 競合は last-write-wins + `_conflict` シート退避
