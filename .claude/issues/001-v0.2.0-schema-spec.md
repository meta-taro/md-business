### B. 章ツリーの扱い

基本設計書は「章・節・項」の階層が肝。選択肢:

- **B1**: 本文の Markdown 見出し（`#` `##` `###`）から自動でツリー生成、左ペインに目次表示
- **B2**: frontmatter `outline:` で章構造を明示宣言、本文は対応する見出しでマッチ
- **B3**: 複数 md ファイル（章ごと .md、`outline:` で順序指定）= API 仕様書と同じディレクトリ方式の前倒し

**推奨**: B1 から始める（v0.2.0 は単一 md 完結、v0.4.0 のディレクトリ基盤待ち）。

### C. 画像 / SVG / Mermaid 対応スコープ

- ✅ 外部参照画像: `![](./figs/x.svg)` ローカル、 `![](https://.../x.png)` HTTPS（Chrome 拡張は CSP 制約あり、`data:` か bundle 内推奨）
- ✅ インライン SVG: 本文中の `<svg>...</svg>` をサニタイズ後そのまま描画
- ✅ Mermaid: ` ```mermaid ` コードブロックを動的 import で描画（初期バンドル肥大化回避）
- ⏸️ draw.io ネイティブレンダリング: 見送り（SVG エクスポート運用で 80% カバー）

CSP / セキュリティ考慮（要 [security-reviewer]）:
- `data:image/{png,jpeg,gif,webp,svg+xml}` のみホワイトリスト（invoice の logo と同方針）
- SVG は `<script>` `on*=` 属性除去
- Mermaid 動的 import は同パッケージ内に bundle 同梱（リモートコード不使用維持）

### D. PDF 出力レイアウト

- A4 縦、表紙 + 目次 + 本文の 3 セクション
- 章タイトル右上にドキュメント番号 + 章番号を running header
- ページ番号は footer
- 画像は中央配置 + キャプション
- Mermaid 図は SVG として埋め込み（PDF 化時もテキスト検索可能）
