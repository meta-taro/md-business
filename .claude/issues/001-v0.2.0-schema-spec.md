### B. 章ツリーの扱い

基本設計書は「章・節・項」の階層が肝。選択肢:

- **B1**: 本文の Markdown 見出し（`#` `##` `###`）から自動でツリー生成、左ペインに目次表示
- **B2**: frontmatter `outline:` で章構造を明示宣言、本文は対応する見出しでマッチ
- **B3**: 複数 md ファイル（章ごと .md、`chapters:` で順序指定）= API 仕様書と同じディレクトリ方式の前倒し

**推奨（2026-06-16 PdM 判断）**: **B1（単一 md）+ B3（複数 md インデックス参照）の併用**。
- 小規模設計書は単一 md で完結、目次は本文の `#` `##` から自動生成（`toc: auto`）
- 大規模化したら `chapters: [./01-overview.md, ./02-architecture.md, ...]` を frontmatter に書き、ビューワーは index.md だけ受け取って参照先を fetch
- API 仕様書議論で確定した「**C 案（インデックス + 明示参照）**」と同じ哲学。ファイル名 prefix で順序制御する A 案は、ツリー記号や 01- 02- 番号管理が人間負担になるため不採用
- B3 を採用する場合、Chrome 拡張は webkitGetAsEntry でディレクトリドロップ対応（実装は v0.4.0 で本格化、v0.2.0 では単一 md 優先）

### C. 画像 / SVG / Mermaid 対応スコープ

- ✅ 外部参照画像: `![](./figs/x.svg)` ローカル、 `![](https://.../x.png)` HTTPS（Chrome 拡張は CSP 制約あり、`data:` か bundle 内推奨）
- ✅ インライン SVG: 本文中の `<svg>...</svg>` をサニタイズ後そのまま描画
- ✅ Mermaid: ` ```mermaid ` コードブロックを動的 import で描画（初期バンドル肥大化回避）
- ⏸️ draw.io ネイティブレンダリング: 見送り（SVG エクスポート運用で 80% カバー）

CSP / セキュリティ考慮（要 [security-reviewer]）:
- `data:image/{png,jpeg,gif,webp,svg+xml}` のみホワイトリスト（invoice の logo と同方針）
- SVG は `<script>` `on*=` 属性除去
- Mermaid 動的 import は同パッケージ内に bundle 同梱（リモートコード不使用維持）

### D. PDF 出力レイアウト

- 画像は中央配置 + キャプション
- Mermaid 図は SVG として埋め込み（PDF 化時もテキスト検索可能）

## 調査結果サマリ（2026-06-14 並行 Explore）

### Markdown ベースの設計書テンプレ比較

| テンプレ | frontmatter | 章数 | 多ファイル | 採否 |
|---|---|---|---|---|
| MADR (Markdown ADR) | あり (status, deciders, date) | 5-6 (1 ADR=1 file) | × | △ 参考 |
| arc42 | なし (AsciiDoc native TOC) | 12 固定章 | ◯ (1 章 1 ファイル) | × prefix 順序問題 |
| Google Design Docs | なし | 8-9 | × | △ 章名は流用 |

**採用方針**: frontmatter は **MADR ライク**（authors / reviewers / status / relatedDocs）、章構成は **Google Design Docs の 8 章ベース**、多ファイル化は **arc42 とは違い prefix ではなく `chapters:` 明示参照**（= C 案哲学）。

### Mermaid / SVG / 画像ライブラリ選定（v0.2.0 範囲）

| ライブラリ | バージョン | 役割 | bundle 影響 |
|---|---|---|---|
| `mermaid` | `10.9.0` | フローチャート描画（動的 import） | +200KB gzip（on-demand） |
| `dompurify` | `3.0.9` | SVG / インライン HTML サニタイズ | +18KB gzip |
| `remark-gfm` | `4.0.0` | GFM 表・チェックボックス | +5KB gzip（既存 remark に追加） |
| 自前 `sanitizeImageUrl()` | - | `data:` + `https:` のみ許可 | 0 |

**CSP**: `script-src 'self'; img-src 'self' data: https:; style-src 'self' 'unsafe-inline'` を `manifest.json` に追加。

**PDF**: Paged.js + Mermaid 描画後の SVG をインライン埋め込み → PDF でもテキスト検索可。
