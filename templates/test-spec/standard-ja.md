# 受発注ワークフロー 検証シート

本シートは基幹システム「Order Console」の受発注ワークフロー機能リリース前検証を目的とした検証シートです。
frontmatter の `列定義` がそのまま Google Sheets の DataValidation / ConditionalFormat / SetFrozenRows に反映され、シート編集後にサイドバーの「GitHub に push」ボタンを押すと md ファイルへ commit されるラウンドトリップが成立します（`git push` と同じメンタルモデル）。

## 検証範囲

- 受注登録（新規 / CSV 一括取込）
- 在庫引当（在庫不足時のエラー / 部分出荷）
- 受注金額計算（消費税 10% / 軽減税率 8% 混在）
- 顧客マスタ連携（未登録 / 取引限度額）
- 出荷指示書 → 納品書 → 請求書の自動連携
- EDI 連携（取引先システムへの受注データ送信）

## メモ

- 与信枠超過の承認待ち遷移が機能していない（NG 行）→ 緊急で別 Issue 起票（与信管理の根幹）
- 請求書メール送信の手動化（保留 行）→ メール送信モジュール改修で対応予定
- 部分出荷の残数繰越（保留 行）→ 月次バッチ実装漏れ、別 Issue 起票
- CSV 一括取込の文字化け（NG 行）→ 文字コード判定ロジックを `juniversalchardet` に置き換え予定
- 軽減税率 / EDI 連携は次フェーズで検証（適格請求書スキーマ + EDI ゲートウェイ確定待ち）
